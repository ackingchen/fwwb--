# 基于无人机实时航拍的多场景目标智能检测与识别系统前端开发文档

## 1. 项目概述

本项目面向无人机实时航拍场景，构建一套集视频输入、目标检测结果展示、结构化信息输出、历史任务管理与性能评估分析于一体的智能检测与识别系统。系统服务于农业巡检、交通监管、应急救援、生态监测、设施巡检等多类业务场景，重点解决航拍视角下小目标、多目标、复杂背景、光照变化及实时处理等问题。

前端系统作为整个平台的人机交互层，主要承担实时视频可视化展示、检测结果交互呈现、任务过程管理、指标统计分析和系统配置控制等职责，用于支撑比赛演示、功能验证与系统完整性展示。

## 2. 建设目标

前端开发目标如下：

- 构建完整、清晰、专业的可视化操作界面
- 支持实时视频流和本地视频文件接入展示
- 支持目标检测结果的动态叠加显示
- 支持结构化检测数据的实时更新与历史回溯
- 支持精度、召回率、mAP、P-R 曲线等评估指标展示
- 支持系统参数配置与模型切换展示
- 保证演示流程顺畅，突出系统工程化与可展示性

## 3. 系统定位

本系统前端定位为“无人机智能巡检与识别可视化平台”，而非传统后台管理系统。其核心价值不在于复杂业务审批流程，而在于将算法能力以直观、可信、专业的方式展示出来，使评委和用户能够快速理解系统的技术能力、实时性表现和场景适应性。

## 4. 用户角色

系统前端主要面向以下角色：

- 演示人员：用于比赛答辩、成果展示、系统讲解
- 算法开发人员：用于联调模型输出、观察检测结果
- 项目成员：用于查看任务记录、调节参数、分析指标
- 潜在业务用户：用于理解系统在真实场景中的应用方式

## 5. 功能需求

### 5.1 实时监控与展示

系统应支持视频输入与实时检测结果展示，包括：

- 接入无人机实时视频流或本地测试视频
- 播放、暂停、停止视频
- 在视频画面中实时叠加目标检测框
- 显示目标类别、置信度、目标编号
- 显示当前帧目标数量、平均置信度、实时 FPS、处理延迟
- 支持场景标签展示，如城市、农田、山地、夜间、雾天等

### 5.2 检测结果详情

系统应支持结构化检测结果查看，包括：

- 当前帧目标明细列表
- 目标类别、坐标、时间戳、置信度等信息展示
- 目标按类别、时间、置信度进行筛选或排序
- 点击目标列表时联动高亮视频中的对应检测框
- 支持原始画面与检测结果画面对比查看

### 5.3 历史任务管理

系统应支持检测任务的历史记录与回溯，包括：

- 展示历史任务列表
- 显示任务名称、执行时间、输入源、场景类型、检测数量、告警数量、任务状态等字段
- 查看单次任务详情
- 支持历史任务结果回放
- 支持任务结果导出

### 5.4 指标分析与评估

系统应支持模型效果和运行效果的图形化分析，包括：

- 展示 Precision、Recall、mAP@0.5 等核心指标
- 展示各类别 AP 柱状图
- 展示 P-R 曲线
- 展示不同场景下的性能对比结果
- 展示 FPS、推理延迟等实时性指标
- 支持以图表形式展示晴天、夜间、雨雾等复杂环境下的检测表现

### 5.5 系统配置

系统应支持基础参数设置，包括：

- 视频源切换
- 模型版本切换
- 置信度阈值设置
- IoU 阈值设置
- 检测类别开关
- 告警开关
- 结果保存开关

### 5.6 结果输出

系统应支持检测结果的结构化输出，包括：

- 表格化展示检测结果
- JSON 格式结果查看
- 任务报告导出
- 截图或视频片段导出

## 6. 非功能需求

### 6.1 易用性

界面结构需清晰，主要功能 3 次点击内可达。首页应具备“一眼看懂系统能力”的展示效果。

### 6.2 实时性

前端页面应能平滑展示实时检测结果，避免明显卡顿。检测框、目标列表、指标信息更新需保持同步。

### 6.3 可演示性

系统要适合比赛答辩展示，主界面视觉风格需专业，关键指标和动态效果应突出。

### 6.4 可扩展性

前端应支持后续接入更多类别、更多场景及更多模型版本，接口结构和组件设计需具备扩展能力。

### 6.5 兼容性

建议优先适配 `1920x1080` 大屏展示，同时兼顾常规笔记本分辨率显示。

## 7. 页面设计方案

### 7.1 首页 / 实时监控大屏

页面定位：系统主展示页面，用于答辩演示和实时监控。

主要内容包括：

- 顶部：系统标题、当前时间、当前场景、模型版本
- 中部：实时视频显示区
- 视频层：检测框、类别标签、置信度标签
- 左侧：设备状态、帧率、延迟、当前目标数、运行状态
- 右侧：实时检测结果列表、告警信息列表
- 底部：播放控制按钮、日志信息、任务状态栏

### 7.2 检测结果详情页

页面定位：查看当前任务或某一帧的结构化检测结果。

主要内容包括：

- 检测画面展示
- 目标明细表格
- 目标筛选区域
- 原图与检测图切换
- 目标点击联动高亮

### 7.3 历史任务页

页面定位：查看过去检测任务及其结果。

主要内容包括：

- 历史任务列表
- 任务筛选条件
- 任务详情入口
- 任务回放与结果导出按钮

### 7.4 指标分析页

页面定位：展示模型效果与系统性能。

主要内容包括：

- 总体指标卡片
- 各类别 AP 柱状图
- P-R 曲线图
- 场景对比图
- FPS 与延迟折线图

### 7.5 系统配置页

页面定位：展示系统可调节参数。

主要内容包括：

- 输入源设置
- 模型选择
- 阈值调节
- 类别勾选
- 告警与保存配置

## 8. 信息架构

建议前端信息架构如下：

- 首页
- 检测详情
- 历史任务
- 指标分析
- 系统配置

该结构层次较浅，适合比赛项目，便于快速演示和开发。

## 9. 前端技术方案

建议技术栈如下：

- 前端框架：`Vue 3`
- UI 组件库：`Element Plus`
- 图表库：`ECharts`
- 状态管理：`Pinia`
- 路由管理：`Vue Router`
- 网络请求：`Axios`
- 视频与框绘制：`HTML5 Video + Canvas` 或定位叠加层
- Mock 调试：`Mock.js` 或本地 JSON 模拟数据

如果团队更熟悉 React，也可以替换为：

- `React`
- `Ant Design`
- `ECharts`
- `Zustand`
- `React Router`

## 10. 数据接口设计建议


```ts
type TaskItem = {

  name: string;
  scene: string;
  source: string;
  createdAt: string;
```

```ts
type MetricSummary = {
```

## 12. 开发优先级

### 第一阶段：核心展示闭环

- 首页大屏
- 视频播放
- 检测框叠加
- 实时目标列表
- 基础指标卡片

### 第二阶段：系统完整性补充

- 历史任务页
- 检测详情页
- 指标分析页
- 配置页基础功能

### 第三阶段：演示效果优化

- 告警动画
- 图表美化
- 原图检测图对比
- 数据导出
- 页面适配优化

## 13. 团队协作建议

前端与后端、算法协作建议如下：

- 算法组负责模型训练、推理输出、指标计算
- 后端组负责视频流接入、任务管理、结果存储、接口封装
- 前端组负责界面设计、交互开发、图表展示、接口联调与系统演示页面实现

前端应尽早与算法和后端统一输出字段，避免后期重复改接口。

## 14. 风险分析

项目实施中可能存在以下风险：

- 算法接口输出不稳定，导致前端联调延迟
- 实时视频流接入复杂度高，影响整体展示
- 数据格式未统一，导致结果展示困难
- 时间紧张，页面过多反而降低完成度

对应策略：

- 前期先使用 Mock 数据开发页面
- 主页面优先开发，确保最小演示闭环
- 提前定义统一数据结构
- 控制页面数量，优先保证质量和稳定性

## 15. 预期成果

前端部分最终应形成以下成果：

- 一套可运行的无人机目标检测前端展示系统
- 一套完整的比赛演示界面
- 支持实时展示、历史回放、指标分析和参数配置
- 具备较强的可视化表达能力和答辩展示效果

## 16. 总结

本前端系统以“实时可视化、结果结构化、指标图形化、系统完整化”为设计原则，围绕比赛题目要求，突出无人机航拍场景下多场景目标检测与识别系统的实时性、准确性、鲁棒性和工程化能力。开发过程中应坚持“主页面优先、演示闭环优先、评分点优先”的原则，确保在有限时间内完成一个可运行、可展示、可答辩的高质量作品。
