## 1. 引言
本文档旨在为无人机目标检测系统提供全面、标准的后端 API 开发参考依据。基于前端工程（Vue 3 + Pinia）的代码深度扫描、业务逻辑分析与数据结构推导，文档覆盖了从视频流接入、目标检测、硬件监控、统计分析到系统日志的**全量接口规范**。

**本文档具有最高优先级，可作为后端开发团队实施 API 设计与开发的唯一依据。**

## 2. 环境配置说明
系统分为三个主要环境，请根据实际部署情况调整域名及配置：
- **开发环境 (DEV)**: `http://dev-api.example.com/api` (用于本地开发与联调)
- **测试环境 (TEST)**: `http://test-api.example.com/api` (用于 QA 测试与自动化测试)
- **生产环境 (PROD)**: `https://api.example.com/api` (必须使用 HTTPS，提供给最终用户)

## 3. 基础规范

### 3.1 API命名规范
严格遵循 **RESTful** 设计原则：
- 使用**名词复数**表示资源，例如 `/users`, `/videos`, `/detections`
- 使用 HTTP 方法表达动作（GET 获取, POST 创建, PUT/PATCH 更新, DELETE 删除）
- 路径中全部使用**小写字母**，单词间使用连字符 `-` 连接
- 避免在路径中使用动词，如不要使用 `/get-video`，应为 `GET /videos`

### 3.2 HTTP状态码使用规范
| 状态码 | 含义 | 适用场景 |
| :--- | :--- | :--- |
| **200 OK** | 请求成功 | 适用于 GET, PUT, PATCH, DELETE 及常规 POST |
| **201 Created** | 资源创建成功 | 适用于 POST 创建异步任务或新资源 |
| **400 Bad Request** | 客户端错误 | 请求参数格式错误、校验失败等 |
| **401 Unauthorized** | 未授权 | 缺少 Token、Token 过期或无效 |
| **403 Forbidden** | 禁止访问 | 用户认证成功但权限不足 |
| **404 Not Found** | 资源不存在 | 请求的 URL 或特定资源 ID 不存在 |
| **422 Unprocessable Entity** | 业务逻辑错误 | 格式正确但业务条件不满足 |
| **500 Internal Server Error** | 服务器内部错误 | 未捕获的代码异常、模型推理宕机等 |

### 3.3 版本控制策略
- **URL 版本控制**：强制将版本号放入 URL 中，例如 `/api/v1/detections`
- **版本弃用策略**：旧版本需至少保留 3 个月的平滑过渡期，并在响应 Header 中添加 `Deprecation` 和 `Link` 提示。

## 4. 安全与认证

### 4.1 认证授权机制详细说明
- **认证方式**：采用 **JWT (JSON Web Token)** Bearer 认证机制。
- **Token 携带**：所有受保护的 HTTP API 必须在 Header 中包含：`Authorization: Bearer <Your-Token>`。
- **WebSocket 鉴权**：在建立连接时的 URL Query 中携带 `?token=<Your-Token>`，或在 `Sec-WebSocket-Protocol` 中传递。
- **Token 续期**：支持无感刷新机制，返回 401 时客户端需使用 Refresh Token 获取新的 Access Token。

### 4.2 数据加密要求
- **传输层**：生产环境强制使用 **TLS 1.2+ (HTTPS/WSS)**。
- **敏感字段**：用户密码必须使用 `Bcrypt` 或 `Argon2` 进行单向哈希存储。

## 5. 全局异常与边界处理

### 5.1 全局错误码对照表
所有失败响应（HTTP 4xx/5xx）需遵循统一的 JSON 结构返回具体的业务错误码：
| 业务错误码 | 错误描述 | 解决方案 |
| :--- | :--- | :--- |
| `ERR_1001` | 参数校验失败 | 检查请求参数类型、必填项及长度是否符合文档要求 |
| `ERR_2001` | Token 无效或已过期 | 重新登录或使用 Refresh Token 换取新 Token |
| `ERR_3001` | 媒体格式不支持 | 确保上传文件为限定格式（如 jpg, png, mp4, avi） |
| `ERR_3002` | 媒体体积超限 | 确保图片 < 10MB，视频 < 500MB |
| `ERR_4001` | AI 模型推理失败 | 检查显存占用，或尝试重启推理服务 |

### 5.2 边界条件与异常处理规范
- **分页边界**：页码 `page` 最小为 1，页大小 `size` 限制在 10~100 之间，超出范围需按默认值处理。
- **并发控制**：对高频请求接口（如图片检测）配置限流（Rate Limiting），触发限流时返回 `429 Too Many Requests`。
- **异常捕获**：服务端应配置全局异常拦截器，禁止向客户端抛出原生报错或 Stack Trace。

## 6. 业务模块 API 详情

### 6.1 视频流与目标检测模块

#### 6.1.1 启动实时流检测 (WebSocket)
- **接口名称**: 实时流检测接入与分发
- **版本号**: v1
- **请求路径**: `ws://<host>/api/v1/stream-detect`
- **支持方法**: WebSocket (Upgrade)
- **业务用途**: 接收前端发送的 RTSP 地址，后端连接摄像头视频流，进行实时 YOLO 推理，并向前端下发 Base64 渲染流与检测框数据。
- **性能要求**: 握手响应 < 200ms；单路流端到端延迟控制在 50ms 内。

**客户端请求消息 (JSON):**
| 字段名 | 必填 | 数据类型 | 说明 | 示例值 |
| :--- | :---: | :--- | :--- | :--- |
| `url` | 是 | String | RTSP/RTMP 直播流源地址 | `"rtsp://192.168.191.60:8080/h264.sdp"` |

**服务端推送消息 (JSON):**
```json
{
  "timestamp": 1682390123.456,
  "image": "/9j/4AAQSkZJRgABAQAAAQABAAD/2wBDAAg... (Base64编码的JPEG流)",
  "detections": [
    {
      "id": "D-01",
      "label": "Person",
      "confidence": 0.94,
      "bbox": [120, 45, 180, 210],
      "level": "danger"
    }
  ]
}
```

#### 6.1.2 上传并检测本地视频
- **接口名称**: 上传本地视频
- **版本号**: v1
- **请求路径**: `/api/v1/detections/video`
- **支持方法**: POST
- **业务用途**: 供用户上传本地视频文件并存储到服务端，返回对应的路径，以便通过 WebSocket 进行同步逐帧推理。
- **性能要求**: 支持分片上传机制，接口响应 < 1s。

**请求参数 (Content-Type: multipart/form-data):**
| 字段名 | 必填 | 数据类型 | 说明 |
| :--- | :---: | :--- | :--- |
| `file` | 是 | File | 视频文件 (MP4/AVI，<500MB) |

**成功响应示例 (200 OK):**
```json
{
  "code": 0,
  "message": "上传成功",
  "data": {
    "videoPath": "/uploads/videos/1489368329-1-100026.mp4"
  }
}
```

#### 6.1.3 视频检测控制与同步 (WebSocket)
- **接口名称**: 本地视频检测同步控制通道
- **版本号**: v1
- **请求路径**: `ws://<host>/api/v1/video-detect`
- **业务用途**: 接收前端发送的视频时间戳，进行关键帧对齐与推理，并将当前帧的检测结果返回前端用于 Canvas 渲染。

**客户端请求指令 (JSON):**
```json
// 启动检测
{ "command": "START", "videoPath": "/uploads/videos/1489368329-1-100026.mp4" }
// 同步播放进度 (心跳)
{ "command": "SYNC_TIME", "currentTime": 12.5 }
// 暂停 / 恢复 / 跳转
{ "command": "PAUSE" }
{ "command": "RESUME" }
{ "command": "SEEK", "seekTime": 25.0 }
```

**服务端推送消息 (JSON):**
```json
{
  "timestamp": 12.5,
  "detections": [
    { "id": "V-01", "label": "Vehicle", "confidence": 0.89, "bbox": [50, 50, 200, 200] }
  ]
}
```

#### 6.1.4 上传并检测本地图片
- **接口名称**: 本地图片目标检测
- **版本号**: v1
- **请求路径**: `/api/v1/detections/image`
- **支持方法**: POST
- **业务用途**: 上传单张图片进行实时的目标检测。
- **性能要求**: 推理响应时间 < 800ms。

**请求参数 (Content-Type: multipart/form-data):**
| 字段名 | 必填 | 数据类型 | 说明 |
| :--- | :---: | :--- | :--- |
| `file` | 是 | File | 图片文件 (JPG/PNG，<10MB) |

**成功响应示例 (200 OK):**
```json
[
  {
    "id": "IMG-01",
    "label": "Vehicle",
    "confidence": 0.88,
    "bbox": [56, 44, 200, 120]
  }
]
```

## 7. 交付与测试支持

### 7.1 Mock 数据生成规则
- **工具建议**：使用 `Mock.js` 或 `Faker.js` 生成规范的测试数据。
- **核心字段规则**：
  - `id`: 使用 UUID v4 生成或自增主键。
  - `label`: 从预设枚举 `["Person", "Vehicle", "Building", "Animal"]` 中随机抽取。
  - `confidence` / `score`: 随机浮点数，保留两位小数，范围 `0.40 ~ 0.99`。
  - `bbox`: 数组 `[x, y, w, h]`，其中坐标基于前端视频画布分辨率动态等比例缩放生成。

### 7.2 Postman 测试集合与依赖图
- **测试集合交付**：开发完成后，必须输出导出版本的 `UAV_Detection_API_Collection.json`。
- **环境配置**：Postman 中需配置 `{{BASE_URL}}` 与 `{{TOKEN}}` 等环境变量。
- **接口依赖关系图 (架构要求)**：
  ```mermaid
  graph TD
    Client[前端应用 Vue3] -->|HTTP REST| APIGateway[API 网关]
    Client -->|WebSocket| StreamService[视频流接入服务]
    APIGateway --> AuthService[鉴权服务 JWT]
    APIGateway --> CoreAPI[核心业务服务]
    StreamService --> InferenceEngine[YOLO 推理引擎 GPU]
    CoreAPI --> InferenceEngine
    CoreAPI --> DB[(MySQL / Redis)]
  ```
